## ① 法令コンプライアンスの穴（インパクト大）

- [x] **年5日の取得義務管理（最優先）** — `implementation-plan.md` Phase 2 で実装済み
  - 年10日以上付与される労働者は「年5日確実に取得」が会社の義務（2019 働き方改革）。
  - 付与日（基準日）起点の1年間で5日消化できているかを `src/lib/leave/annual-obligation.ts` で社員ごとに追跡。
  - 未達社員を社員一覧・社員詳細で色分け・バッジ表示。「あと◯日 / 期限◯月」の残り義務日数を表示。

- [x] **年次有給休暇管理簿の出力** — Phase 3 で実装済み
  - 労基法で作成・保存が義務（基準日・取得日数・取得時季を記録）。保存期間は原則5年だが、経過措置により当分の間は3年（Codexレビューで指摘・訂正）。
  - `src/lib/leave/ledger-excel.ts` + `src/app/api/employees/[id]/leave-ledger/route.ts` でExcel出力。社員詳細から年度指定でダウンロード可能。

- [x] **時間単位年休への対応** — Phase 4 で実装済み
  - 労使協定がある前提で年5日（40時間）まで時間単位取得が可能。
  - `LeaveUnit` に `hourly` を追加し `LeaveRequest.hours` で保持。所定労働8時間固定、`LeaveGrant.grantedDays`/`LeaveConsumption.consumedDays` は `Decimal(6,3)` に拡張。

## ② 既知の未実装（CLAUDE.md で宣言済み）

- [x] **有給自動付与バッチ** — Phase 1 で実装済み
  - 管理者の手動実行ボタン（`AutoGrantPanel`、プレビュー→確定の2段階）と、Vercel Cron による日次ジョブ（`src/app/api/cron/auto-grants/route.ts`）の両方に対応。
  - `getNextGrantMilestone` を含む `schedule.ts` の純粋関数群と、部分ユニーク制約によるべき等性で二重付与を防止。

- [ ] **出勤率8割要件の運用フロー**
  - `implementation-plan.md` Phase 1 の論点整理で「案A（付与を先に作り、記録欄は後から人事が埋める。残高には即カウント）」に確定し、承認ステップ化（確認待ちリスト等）は現行スコープでは見送りとした。出勤率不良が事後判明した場合は既存の「取消」導線で個別対応する運用。実運用で頻発するようなら改めて検討する。

## ③ 運用を楽にする機能

- [ ] **通知・アラート**（現仕様ではスコープ外）
  - 承認待ちの滞留 / 失効間近（use-it-or-lose-it） / 年5日未達 の3種をメール・Slack通知。
  - まずはアプリ内のお知らせ枠から着手すると軽い。

- [x] **会社カレンダー / 休日マスタ** — 実装済み（2026-07-23）
  - 祝日・会社独自の休日（`Holiday`モデル、`type`で区別）を管理者が手動登録・編集・削除できる（`src/app/holidays/`）。内閣府公表の祝日CSV取り込み（プレビュー→確定の2段階）にも対応（`src/lib/holidays/national-holiday-source.ts`）。
  - 休日マスタに登録された日への有給申請は、単日・期間一括申請とも作成時・承認時の両方でサーバー側ブロックする（警告のみではなく拒否。`TargetOnHolidayError`）。期間一括申請には「休日を除外」チェックボックス（デフォルトON、`skipWeekends`と並置）を追加し、休日を跨ぐ一括申請自体が起きにくい設計にした。

- [x] **CSV / Excel エクスポート** — 実装済み（2026-07-23）
  - 全社員(在職中・退職済み含む)を対象に、指定期間の付与・消化・期末残高をエクスポート(`src/app/employees/export/`, `src/app/api/employees/export/`)。Excelはサマリー/付与明細/消化明細の3シート、CSVは粒度ごとに個別ダウンロード。期末残高は既存の年次有給休暇管理簿と同じ`reviewedAt`基準の厳密なロジックを再利用。

## ④ UX

- [x] **管理者ダッシュボード** — 実装済み(2026-07-24)
  - 承認待ち件数 / 今月失効する有給日数(暦月) / 年5日取得義務の未達人数 / 全社の有給取得率(直近12ヶ月の消化率)を`/dashboard`に集約(`src/app/dashboard/`, `src/lib/dashboard/queries.ts`)。社員一覧のヘッダーからリンク。

- [ ] **チーム休暇カレンダー**
  - 誰がいつ休むかを月表示。休みの集中を管理者が把握できる。

- [x] **モバイル最適化** — 実装済み（2026-07-24）
  - ヘッダーのナビゲーションを`AppHeader`にロール別グローバルナビとして集約し、`lg`未満ではハンバーガーメニューに切り替え(`src/components/nav-menu.tsx`)。社員一覧・社員詳細・承認待ち一覧・特別休暇・監査ログの各テーブルは`md`未満でカード表示に変換。承認/却下/取消ボタンのタッチターゲット拡大、休日マスタのカレンダーセルサイズ調整も実施。

- [x] **残日数の内訳表示** — 実装済み
  - 社員詳細画面の「有給付与履歴」テーブルで、付与枠ごとの残日数・失効予定日・失効間近バッジを表示済み。

- [x] **特別休暇（慶弔・産前産後・育児・夏季休暇）の管理** — 実装済み（2026-07-24）

## ⑤ 技術的な足回り

- [x] **自動テスト** — Phase 0 で導入、以降の全フェーズで拡充済み
  - FEFO消化・残高計算・退職時の自動却下/自動取消は境界条件が多くバグの温床。
  - 純粋関数（`schedule.ts` / `balance.ts` / `request-rules.ts` / `annual-obligation.ts` / `date-range.ts` 等）の単体テストと、mutations/queries の統合テスト（`vitest`）で網羅。

- [x] **監査ログ** — 実装済み（2026-07-23）
  - 有給申請の承認/却下/取消/取り下げ、退職処理、社員登録・編集を対象に、実行者・対象社員・操作内容(`AuditLog`モデル)を記録。各mutation関数の`$transaction`内で記録するため、操作とログの整合性を保証(ロールバック時はログも一緒に消える)。
  - 管理者専用の一覧画面(`src/app/audit-logs/`)で期間・対象社員による絞り込みが可能。
